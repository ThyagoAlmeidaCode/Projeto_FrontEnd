Pontos de Quebra (Breakpoints) Comuns

Breakpoints são valores de largura de tela onde o layout do site se adapta para proporcionar uma melhor experiência do usuário em diferentes dispositivos.

Redimensione a janela para ver o breakpoint atual

Dica: Os breakpoints devem ser baseados no conteúdo, não necessariamente em dispositivos específicos. Use esses valores como referência, mas adapte conforme a necessidade do seu layout.

Breakpoints mais utilizados:

/* Dispositivos muito pequenos (celulares em portrait) */
@media (max-width: 575.98px) { ... }

/* Dispositivos pequenos (celulares em landscape) */
@media (min-width: 576px) and (max-width: 767.98px) { ... }

/* Dispositivos médios (tablets) */
@media (min-width: 768px) and (max-width: 991.98px) { ... }

/* Dispositivos grandes (desktops) */
@media (min-width: 992px) and (max-width: 1199.98px) { ... }

/* Dispositivos extra grandes (desktops grandes) */
@media (min-width: 1200px) { ... }

Estratégias de desenvolvimento:

Mobile First: Desenvolva primeiro para dispositivos móveis e use min-width para adicionar estilos para telas maiores.

Desktop First: Desenvolva primeiro para desktop e use max-width para adaptar para telas menores.

Propriedades Flexbox: flex-grow, flex-shrink e flex-basis

O Flexbox é um modelo de layout unidimensional que facilita o design de estruturas responsivas. As três propriedades a seguir controlam como os itens flexíveis se comportam dentro de um container.

flex-basis

Define o tamanho inicial de um item flexível antes que o espaço restante seja distribuído. Pode ser um valor em px, %, ou auto.

flex-grow

Define a capacidade de um item flexível crescer se necessário. É um número sem unidades que serve como proporção. O valor padrão é 0.

flex-shrink

Define a capacidade de um item flexível encolher se necessário. Também é um número sem unidades que serve como proporção. O valor padrão é 1.

Nota: A propriedade shorthand flex aceita três valores: flex-grow, flex-shrink e flex-basis. Exemplo: flex: 1 0 100px;
